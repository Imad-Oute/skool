## Part 1 — Macro Architecture: The Agency as a Machine

The agency is a machine with three core loops. Every department, every system, every tool exists to serve one of these loops:

```
INPUTS                         MACHINE                          OUTPUTS
─────────────────────────────────────────────────────────────────────────
Prospects      →     [ ACQUISITION LOOP ]      →     Signed Clients
Signed Clients →      [ DELIVERY LOOP ]        →     Results + Revenue
Results        →      [ RETENTION LOOP ]       →     Recurring Revenue + Referrals
                              ↑
                   Powered by: OPERATING SYSTEM (Tier 4)
                   Directed by: CEO LAYER (Tier 1)
```

### The Three Core Loops

**Loop 1 — Acquisition:** How strangers become paying clients
**Loop 2 — Delivery:** How clients get the results they paid for
**Loop 3 — Retention:** How one-time clients become long-term revenue

All three loops fail without the Operating System underneath and the CEO Layer above.

### TIER 1 — CEO Layer

**Owner:** Imad
**Purpose:** Set direction, make key decisions, ensure all tiers operate toward the same goal.

#### Systems Inside Tier 1

| System | What It Is | Cadence | Tool |
|--------|-----------|---------|------|
| Vision & North Star | The 1-year goal broken into 90-day OKRs | Quarterly | Document |
| OKR Tracking | Objectives + Key Results per quarter, reviewed monthly | Monthly | Notion / AppFlowy |
| Financial Dashboard | Revenue, expenses, margins, MRR, client LTV | Weekly | Wave + Spreadsheet |
| Business Development | New client pipeline, BD activities, partnership targets | Ongoing | CRM |
| Key Client Relationships | Direct relationship management for top clients | Weekly | Client Protocol |
| Culture & Standards | Defines what "elite quality" means for every department | Define once, enforce always | Document |
| CEO Weekly Review | Sunday 30-min ritual: review → priorities → decisions | Weekly | Written doc |

#### CEO Operating Rhythm

```
SUNDAY (30 min)
└── Review: What shipped last week?
└── Assess: Are we on track with OKRs?
└── Decide: What's the #1 priority next week?
└── Write: Document decisions and directives

MONDAY (15 min)
└── Communicate priorities to partner via standup

DAILY (15 min)
└── Morning standup: 3 MITs per person
└── Evening pulse: Did we hit our MITs?

FRIDAY (30 min)
└── Review: What shipped this week?
└── Client: Send all client weekly updates
└── Retrospect: What broke? What worked?
```

## Part 4 — The Full Agency Workflow (Macro to Micro)

### Macro Workflow

```
PROSPECT → CLIENT → DELIVERING → RESULTS → RETAINED → REFERRAL
```

### Mid-Level Workflow

```
ACQUISITION PHASE
└── BD Activity → Lead Identified → Discovery Call → Proposal → Close → Contract

ONBOARDING PHASE
└── Contract Signed → Kickoff → Project Setup → Access Granted → Work Begins

DELIVERY PHASE (per service, runs in parallel)
└── Audit/Discovery → Strategy → Production → QA → Client Review → Launch → Monitor → Report

OPTIMIZATION PHASE (runs continuously)
└── Data → Insights → Adjustments → New Strategy → Repeat

RETENTION PHASE
└── Monthly Review → Results Presented → Upsell / Expand → Referral Ask
```

### Micro Workflow (Step-by-Step, End to End)

```
STEP 1: BD & LEAD GENERATION
├── Owner: Imad (CEO)
├── Activities: Outbound outreach, referral follow-up, inbound content
├── Output: Qualified prospect in CRM
└── Gate: Is this a client we can deliver results for?

STEP 2: DISCOVERY CALL
├── Owner: Imad
```
