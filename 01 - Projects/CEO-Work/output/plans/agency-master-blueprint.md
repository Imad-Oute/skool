## 2.2 CEO Operating Systems

### Vision & OKRs
```
STRUCTURE
├── Annual Vision: The 1-year goal (where are we by April 2027?)
├── Quarterly OKRs (3-month objectives + measurable key results)
│   ├── O1: [Objective] — What we're trying to achieve
│   │   ├── KR1: [Key Result] — Measurable outcome
│   │   ├── KR2: [Key Result]
│   │   └── KR3: [Key Result]
│   └── O2: [Objective]
├── Monthly Targets: Break each KR into monthly milestones
└── Weekly Priorities: 3 MITs per week that move the needle on KRs

TOOL: Obsidian (desktop) — /Areas/Agency-CEO/OKRs/
CADENCE: Set quarterly → review monthly → execute weekly
```

### Financial Dashboard
```
WHAT TO TRACK (weekly)
├── Revenue: All invoices sent this month
├── Collected: Cash actually received
├── Outstanding: Invoices unpaid (chase these)
├── MRR: Monthly recurring (retainer clients)
├── Expenses: Every tool, subscription, service paid
├── Net Margin: Revenue - Expenses = actual profit
├── Client LTV: Average revenue per client per month
└── Pipeline Value: Total value of open proposals

TOOL: Invoice Ninja (Docker) → revenue + invoices
      Obsidian spreadsheet or LibreOffice → P&L tracker
CADENCE: Update weekly (Friday)
```

### CEO Weekly Review (Sunday — 30 min, non-negotiable)
```
AGENDA (written in Obsidian weekly note)
├── 1. REVIEW (10 min)
│   ├── What shipped this week?
│   ├── What didn't ship? Why?
│   ├── Revenue/pipeline update
│   └── Client health status
├── 2. ASSESS (10 min)
│   ├── Are we on track for this quarter's OKRs?
│   ├── What's the biggest risk right now?
│   └── What decision have I been avoiding?
└── 3. DIRECT (10 min)
    ├── What's the #1 priority next week?
    ├── What are the 3 MITs?
    └── What needs to be decided or communicated?
```

```
06:00  Wake + Exercise (30 min minimum — non-negotiable for cognitive performance)

       → Open yesterday's note → review
```

## Department 1: Paid Ads

**Owner:** Imad / Partner
**Goal:** Generate qualified leads for clients at target CPL

### Tools
```
Google Ads              → Campaign management (required — no alternative)
Google Ads API          → Automation (create_rolland_campaigns.py)
Google Analytics 4      → Conversion tracking (required)
Google Tag Manager      → Tag deployment (required)
Google Search Console   → Search performance (free)
Google Looker Studio    → Client dashboards (free)
n8n (Docker)            → Campaign automation + reporting workflows
```

### Delivery Pipeline
```
STEP 1: CLIENT BRIEF
├── Input: Client goals, budget, target audience, geography, KPIs
├── Tool: Formbricks questionnaire → responses in Obsidian
└── Output: Completed brief doc

STEP 2: MARKET & COMPETITOR AUDIT
├── What are top competitors running? (SpyFu free tier)
├── What keywords are they targeting?
├── What ad copy angles are working?
└── Output: Competitor intel doc in Obsidian

STEP 3: CAMPAIGN STRATEGY
├── Keyword research (Google Keyword Planner + Search Console data)
├── Campaign structure (campaigns, ad groups, keywords)
├── Budget allocation per campaign
├── Bidding strategy recommendation
├── Ad copy angles (minimum 3 per ad group)
└── Output: Written strategy doc → client approval required before Step 4

STEP 4: CREATIVE BRIEF → DESIGN DEPT
├── Ad copy variations (3+ per ad group)
├── Landing page requirements (if new LP needed)
└── Output: Brief handed to Creative & Design dept

STEP 5: CAMPAIGN SETUP
├── Account structure: Campaign → Ad Group → Keywords → Ads
├── Keyword match types configured
├── Negative keyword list added (minimum 50 keywords)
├── Ad extensions set up (sitelinks, callouts, call extension)
├── Budget + bid strategy configured
└── Output: Campaign built, paused (NOT live until QA passes)

STEP 6: TRACKING SETUP
├── Conversion action created in Google Ads
├── GTM container installed on site
├── Conversion tag deployed via GTM
├── GA4 ↔ Google Ads linked
├── Test conversions fire correctly (mandatory)
└── Output: Tracking verified ✅

STEP 7: QA GATE (nothing goes live until all pass)
├── □ Conversion tracking fires correctly (test in real browser)
├── □ All ad copy reviewed + approved by Imad
├── □ Negative keyword list added
├── □ Daily budget cap set
├── □ Geographic targeting confirmed
├── □ Ad schedule set (if applicable)
├── □ Landing page loads correctly + mobile-friendly
└── □ Client has given written approval

STEP 8: LAUNCH
├── Campaigns set to Active
├── Monitor first 4 hours manually
└── Screenshot of launch state → save to client folder

STEP 9: WEEK 1 MONITORING (daily)
├── Check: Impressions, clicks, CTR, conversions, budget pacing
├── If CTR < 2%: Pause low performers, test new ad copy
├── If budget over-pacing: Reduce bids
├── If no conversions after 50 clicks: Check tracking + landing page
└── Notes in Plane (issue per observation)

STEP 10: WEEKLY OPTIMIZATION
├── Search term report → add converting terms as exact match, negatives
├── Bid adjustments (device, time of day, geography)
├── A/B test: pause worst-performing ad, add new variant
├── Quality Score review (target > 7)
└── Update client via weekly report (Looker Studio dashboard)

STEP 11: MONTHLY STRATEGY REVIEW
├── Compare results vs. KPIs from brief
├── What's working? (scale it)
├── What's not? (cut or test new approach)
├── Propose next month's strategy adjustment
└── Present to client in monthly review call

IMPROVEMENT LOOP
├── Weekly: Optimization actions documented in Plane
├── Monthly: Strategy review + client call
├── Quarterly: Full account audit → restructure if needed
└── Library: Save best-performing ad copy to Obsidian /Resources/What-Works/Ads/
```

### Quality Standards
```
CTR (Search):          > 5% (under 2% = immediate action)
Quality Score:         > 7/10 per keyword
Conversion Tracking:   100% verified before launch
Budget Pacing:         Within ±10% of daily target
CPL Target:            Defined per client in brief → tracked weekly
Report Delivery:       Every Monday by 10am
```

## Department 2: Content & SEO

**Owner:** Imad / Partner
**Goal:** Rank client pages for target keywords, drive organic leads

### Tools
```
Ahrefs Starter (€29/mo)   → Keyword research + rank tracking + backlink analysis
Google Search Console      → Real ranking data (free, mandatory)
Screaming Frog (free)      → Technical SEO audits (up to 500 URLs)
SurferSEO (optional)       → On-page optimization scoring
Google PageSpeed Insights  → Core Web Vitals (free)
CMS (client's platform)    → WordPress / Webflow / custom
Obsidian                   → Content briefs, editorial calendar
```

### Delivery Pipeline
```
STEP 1: SEO AUDIT
├── Technical audit (Screaming Frog): broken links, missing tags, redirects
├── On-page audit: title tags, meta descriptions, H1 structure
├── Performance audit: Core Web Vitals (PageSpeed Insights)
├── Current rankings: GSC data (what pages rank for what?)
└── Output: Audit report with prioritized fix list

STEP 2: KEYWORD RESEARCH
├── Seed keywords from client brief
├── Expand: Ahrefs → Keyword Explorer → related terms
├── Filter: KD < 30 for new sites, volume > 50/month
├── Intent mapping: informational / commercial / transactional
├── Competitor gap: what do they rank for that client doesn't?
└── Output: Keyword matrix (keyword, volume, KD, intent, target page)

STEP 3: CONTENT STRATEGY
├── Pillar pages (broad topic, 2,500+ words)
├── Cluster pages (specific subtopic, 1,200+ words, links to pillar)
├── Content calendar (what publishes when, priority order)
└── Output: Content strategy doc + editorial calendar in Obsidian

STEP 4: CONTENT BRIEF (per article)
├── Target keyword (primary + 3-5 secondary)
├── Search intent (what does the reader actually want?)
├── SERP analysis (top 5 ranking pages — what do they cover?)
├── Article structure (H1, H2, H3 outline)
├── Word count target
├── Internal links to include
├── CTA at end of article
└── Output: Brief in Obsidian /Projects/[Client]/Content/Briefs/

STEP 5: CONTENT PRODUCTION
├── Write from brief (not freeform — follow the brief)
├── Minimum word count hit
├── All H2s/H3s from outline present
├── Primary keyword in: title, H1, first 100 words, meta description
├── Secondary keywords distributed naturally
└── Output: Draft in Obsidian or Google Docs

STEP 6: SEO OPTIMIZATION
├── On-page: title tag (< 60 chars), meta description (< 160 chars)
├── Images: compressed + alt text on all
├── Internal links: minimum 3 per article
├── Schema markup: Article schema if applicable
└── Output: Optimized content ready for QA

STEP 7: QA GATE
├── □ Keyword in title, H1, first paragraph, meta
├── □ Word count hits target
├── □ All images have alt text
├── □ Minimum 3 internal links added
├── □ No grammatical errors (LanguageTool check)
├── □ CTA present at end
└── □ Preview on mobile looks correct

STEP 8: PUBLISH + TECHNICAL
├── Publish to CMS
├── Canonical URL set
├── Submit URL in Google Search Console → Request Indexing
└── Add to rank tracking in Ahrefs

STEP 9: MONTHLY RANKING REVIEW
├── Check target keywords in GSC + Ahrefs
├── Positions improving? → continue, maybe build internal links
├── Positions flat? → update content, add more depth
├── Positions dropping? → investigate (algorithm update? competitor?)
└── Report to client in monthly review

IMPROVEMENT LOOP
├── Monthly: Rank tracking review per article
├── Quarterly: Content audit (update, merge, or delete underperformers)
├── Quarterly: Keyword gap analysis (new opportunities)
└── Library: Save best-performing content formats to Obsidian
```

### Quality Standards
```
Minimum word count:     1,200 words (pillar: 2,500+)
Keyword placement:      Title + H1 + first 100 words + meta
Internal links:         Minimum 3 per article
Image optimization:     All images < 200KB, alt text on all
Core Web Vitals:        LCP < 2.5s, CLS < 0.1 (green)
Publishing schedule:    On-time per editorial calendar (no delays)
```

## Department 3: Web Development

**Owner:** Imad / Partner
**Goal:** Build fast, SEO-optimized, high-converting websites

### Tools
```
Penpot (Docker)           → UI/UX design, wireframes, mockups
VS Code                   → Code editor (primary)
Next.js / Astro           → Frontend framework (project-dependent)
GitHub                    → Version control
Coolify (future)          → Deployment (when server ready)
Google PageSpeed          → Performance testing
Screaming Frog            → Post-launch SEO audit
GTmetrix                  → Performance benchmarking
```

### Technology Decision Framework
```
USE ASTRO when:
├── Content-heavy site (blog, news, documentation)
├── Maximum SEO performance needed
├── Minimal JavaScript interactivity
└── Static or near-static content

USE NEXT.JS when:
├── Dynamic content (user accounts, dashboards)
├── E-commerce
├── Requires server-side rendering or API routes
└── Complex interactivity

USE WORDPRESS when:
├── Client wants to edit own content (non-technical)
├── Existing WordPress site being redesigned
└── Budget is limited + fast delivery needed
```

### Delivery Pipeline
```
STEP 1: DISCOVERY & BRIEF
├── Goals: What should the site achieve? (leads, sales, brand)
├── Audience: Who visits? What do they need?
├── Pages needed: Sitemap
├── Brand assets: Logo, colors, fonts, images
├── References: 3 sites client likes + why
├── Technical: Current hosting, domain, CMS preference
└── Output: Project brief + signed off by client

STEP 2: SITE ARCHITECTURE
├── Sitemap (all pages, hierarchy)
├── URL structure
├── Navigation structure
├── Page priority for SEO
└── Output: Sitemap doc in Plane

STEP 3: WIREFRAMES (Penpot)
├── Low-fidelity layout for each unique page type
├── Mobile-first (design mobile before desktop)
├── Focus on: information hierarchy, CTA placement
└── Output: Wireframes in Penpot → client review

STEP 4: HIGH-FIDELITY DESIGN (Penpot)
├── Apply brand: colors, typography, imagery
├── Design all components (buttons, forms, cards, nav)
├── Design all page types (home, service, contact, landing)
├── Desktop + mobile versions
└── Output: Full Figma/Penpot file → CLIENT APPROVAL required

STEP 5: DEVELOPMENT
├── Set up project repo on GitHub
├── Initialize framework (Next.js / Astro / WP)
├── Build component library first (reusable components)
├── Build pages from approved designs
├── Implement SEO: meta tags, schema, sitemap.xml, robots.txt
├── Forms: set up contact/lead forms with email notifications
└── Progress tracked in Plane (one issue per page/component)

STEP 6: CONTENT INTEGRATION
├── All copy added (client-provided or written by Content dept)
├── All images optimized (WebP format, compressed, lazy loaded)
├── Videos embedded correctly
└── All placeholder content replaced

STEP 7: PERFORMANCE OPTIMIZATION
├── Images: WebP, correct dimensions, lazy loading
├── Fonts: self-hosted or display:swap

├── □ GA4 + GTM tracking fires correctly
├── □ All links work (no broken links)
└── □ ORIAS/legal requirements on page (if applicable)

STEP 9: CLIENT REVIEW (UAT)
├── Share staging URL
├── Record Loom video walkthrough of the site
├── Client tests all functionality
├── One round of revisions included
└── Written approval before going live

STEP 10: LAUNCH
├── DNS pointed to hosting
├── SSL active + tested
├── 301 redirects set up (if redesign)
├── Google Search Console: submit sitemap
├── Google Ads: update landing page URLs if needed
```

## 8.3 Service-by-Service Configuration

### Plane Configuration
```
1. Open http://localhost:3000
2. Create admin account (use email + strong password → save in KeePassXC)
3. Create Workspace: "[Your Agency Name]"
4. Create Projects:
   - "RA-Project" (RollandAssurance)
   - "Agency-Internal" (systems, operations, BD)
5. In RA-Project → create Modules:
   - Website
   - Paid Ads
   - Content & SEO
   - Lead Gen Automation
6. Set up Labels: Urgent / High / Medium / Low / Bug / Feature
7. Create first Sprint (Cycle): "Sprint 1 — RA Foundation Gates"
8. Add all 9 foundation gate tasks from RA-Project README
```

### n8n Configuration
```
1. Open http://localhost:5678
2. Login with N8N_USER + N8N_PASSWORD from .env
3. Create credentials (saved once, used in all workflows):
   - Google Ads API (OAuth2)
   - Gmail / SMTP (for notifications)
   - Airtable API (for RA lead gen MVP)
   - OpenAI API key
4. Import existing workflows:
   - Upload RA-Project n8n blueprints from /05-Client-OS/RA-Project/output/plans/
5. Test each workflow manually before enabling schedules
```

### Ollama Configuration
```bash
# Pull models after Ollama starts

# Verify models loaded
```

### Open WebUI Configuration
```
1. Open http://localhost:8080
2. Create admin account
3. Verify Ollama connection: Settings → Connections → Ollama Base URL = http://ollama:11434
4. Create system prompts:
   - "Agency Assistant": knows about your agency, services, pricing
   - "Code Reviewer": reviews Python/JS code for quality + security
   - "Content Writer": writes in your brand voice
5. Test each model works: mistral, codellama
```

### Uptime Kuma Configuration
```
1. Open http://localhost:3002
2. Create admin account
3. Add monitors for every running service:
   - Plane: HTTP(s) → http://localhost:3000 → every 60s
   - n8n: HTTP(s) → http://localhost:5678 → every 60s
   - Open WebUI: HTTP(s) → http://localhost:8080 → every 60s
   - Ollama: HTTP(s) → http://localhost:11434 → every 60s
4. Add notification: Email or Telegram on down
5. Add external monitors (client sites):
   - RA-Project website (when live)
```

# PART 9 — SCALING PATH

## When to Deploy to a Server

```
TRIGGER: Deploy to VPS when any of these are true:
├── You need n8n running 24/7 for client automations that run overnight
├── You need to give clients access to dashboards/portals
├── Your laptop being off breaks a client deliverable
└── You're spending > 30 min/week on local service issues

PROCESS:
1. Get Hetzner VPS (CAX21: €8/month, 4 vCPU ARM, 8GB RAM)
2. Install Coolify on the VPS (open-source deployment platform)
3. Push agency-infra repo to VPS
4. Run same docker-compose.yml (with production .env)
5. Point domains via Cloudflare DNS
→ Zero migration. Same compose file. Done.
```
