# Cross-Domain Synthesis Lenses

Use these lenses to produce a structurally sound synthesis in step-03. Choose the lens that matches what you extracted from Domain A.

---

## Lens 1: Structural Isomorphism

**Use when:** Domain A solves a problem using the same underlying causal structure as Domain B's problem — even though the surface looks nothing alike.

**Method:** Identify the abstract causal loop in Domain A (input → mechanism → output → feedback). Map each element to its functional equivalent in Domain B. The synthesis is valid if the mechanism transfers — not just the output.

**Example:** Immune system memory (Domain A) → Product onboarding re-engagement (Domain B). Both solve the problem of recognizing a known entity at second contact without repeating the full initial process.

---

## Lens 2: Constraint Transfer

**Use when:** Domain A operates under a constraint that forced an elegant solution — and Domain B faces the same constraint but hasn't been forced to solve it yet.

**Method:** Identify the binding constraint in Domain A. Verify it applies (or will apply) in Domain B. The synthesis is the Domain A solution adapted to Domain B's specifics.

**Example:** Formula 1 pit stop efficiency (Domain A: extreme time constraint) → Emergency room patient handoffs (Domain B: same time pressure, different context). Same constraint → similar solution structure.

---

## Lens 3: Failure Mode Transfer

**Use when:** Domain A has well-documented failure modes from long operational experience — and Domain B is newer and hasn't encountered them yet.

**Method:** Catalog Domain A's top failure modes. Test whether each could manifest in Domain B. The synthesis is a pre-emptive design change in Domain B based on Domain A's hard-won lessons.

**Example:** Aviation safety culture (cockpit resource management, checklists) → Hospital surgical teams. Same failure modes (authority gradient, communication breakdown) in different contexts.

---

## Lens 4: Mechanism Reuse

**Use when:** Domain A uses a specific mechanism that solves a class of problems — and Domain B has a problem in that class that it's solving poorly.

**Method:** Abstract the mechanism from Domain A (not the implementation — the underlying principle). Identify how it would be implemented in Domain B's context.

**Example:** Netflix's recommendation engine mechanism (collaborative filtering) → B2B sales outreach prioritization. Same class of problem: ranking a large item set by predicted relevance to a specific recipient.

---

## Synthesis Validity Test

Before presenting the synthesis, verify:

1. **Not surface similarity** — the analogy isn't based on vocabulary overlap
2. **Mechanism transfers** — the causal mechanism actually works in Domain B
3. **Breaks existing thinking** — a domain expert would find this non-obvious
4. **Produces a decision** — the synthesis suggests at least one concrete design choice
