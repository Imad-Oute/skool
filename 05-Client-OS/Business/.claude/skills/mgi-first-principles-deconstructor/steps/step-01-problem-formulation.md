# Step 1: Problem Formulation

**Agent:** Thinker 🧠  
**Objective:** State the problem precisely. Audit which constraints are real vs. inherited.

---

## Your Task

You are the **Thinker**. Before deconstructing anything, the problem must be stated with axiomatic precision. Vague problems produce vague axioms.

### 1. Problem Statement

Answer each question with maximum specificity:

```
What are you trying to achieve?    _______________________________________________
What is currently preventing it?   _______________________________________________
How long has this been a problem?  _______________________________________________
What solutions have failed?        _______________________________________________
Why did they fail?                 _______________________________________________
```

### 2. Constraint Audit

List every constraint, then classify each one:

| Constraint   | Type                                                                                           | Evidence                       |
| ------------ | ---------------------------------------------------------------------------------------------- | ------------------------------ |
| [constraint] | Physical Law / Logical Necessity / Industry Convention / Inherited Assumption / Resource Limit | [why you believe this is real] |

**Classification rules:**

- **Physical Law**: violating it is physically impossible (gravity, thermodynamics, speed of light)
- **Logical Necessity**: violating it creates a logical contradiction
- **Industry Convention**: "this is how it's done" — potentially challengeable
- **Inherited Assumption**: you assumed this from a previous context — needs verification
- **Resource Limit**: real but potentially negotiable

### 3. Problem Redefinition Test

Apply the "5 Whys" to the problem statement:

```
Why is [problem] happening?           → Because: _______________
Why is [that] happening?              → Because: _______________
Why is [that] happening?              → Because: _______________
Why is [that] happening?              → Because: _______________
Why is [that] true?                   → Because: _______________
```

After 5 whys: is the original problem statement still correct? Or has the real problem revealed itself?
