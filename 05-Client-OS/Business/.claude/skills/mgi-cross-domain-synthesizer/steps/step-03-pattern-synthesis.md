# Step 3: Pattern Synthesis

**Agent:** Brainstormer 💡 + Analyst 📊  
**Objective:** Extract the transferable pattern. Generate the novel insight.

---

## Your Task

**Brainstormer** leads, **Analyst** validates. You have a structural mapping. Now synthesize the insight — the thing that Domain A reveals about Domain B that no one inside Domain B would have seen.

### The Synthesis Step

**Brainstormer: Generate the insight**

What does Domain A's mechanism suggest about how Domain B's problem should be approached?

Complete this synthesis statement:

> "Domain A [domain] solves [challenge type] through [mechanism]. Applied to Domain B [your problem], this suggests [novel approach / structural principle]."

State it three ways — get progressively more specific:

```
Version 1 (abstract): _______________
Version 2 (applied):  _______________
Version 3 (specific): _______________
```

**Analyst: Validate the insight**

- Does the insight actually follow from the structural mapping?
- Is there a simpler way to state it?
- What is the falsifiable prediction this insight makes? (If we apply this, what should we observe?)

### Transferable Pattern

Distill the synthesis into a reusable pattern:

```
PATTERN NAME:           _______________
DOMAIN OF ORIGIN:       _______________
CORE MECHANISM:         _______________
APPLICATION CONDITION:  (when is this pattern applicable?)
INAPPLICABILITY:        (when does it NOT apply?)
NOVEL INSIGHT:          (what does this reveal that wasn't visible before?)
```

### Competitive Advantage Assessment

Why wouldn't anyone inside Domain B have found this?

- What domain-local thinking would have prevented it?
- How long would it have taken to find this via conventional approaches?
- What does this unlock that was previously blocked?
